01 项目概述

1.1 项目背景

健身行业正经历数字化转型，但大多数私人教练仍依赖纸质记录、Excel 表格和碎片化的微信沟通管理学员。一位教练通常同时带 20-50 名学员，面临以下痛点：

  • 信息分散：学员体测数据、训练计划、饮食记录散落在不同平台
  • 跟进低效：无法及时追踪学员打卡情况，流失风险识别滞后
  • 服务同质化：难以针对每位学员的身体条件提供个性化方案
  • 数据孤岛：训练数据、饮食数据、体重变化无法联动分析

1.2 解决方案

ClawFit 是一款基于 AI 的智能健身教练助手，以飞书为载体，通过自然语言交互为健身教练提供学员管理、计划生成、打卡跟踪、饮食分析、运营洞察的一站式解决方案。

03 教练端功能详解

3.1 学员建档系统

ClawFit 支持三种建档方式，适配不同场景需求：

💬

对话式建档

发送「新学员」，通过自然对话逐步收集信息，自动生成档案

📄

模板导入

下载 Markdown 模板，离线填写后上传，系统自动解析

📊

批量导入

CSV 模板批量填写，一次性导入多位学员，适合大规模迁移

档案信息维度

维度具体字段用途
基础信息姓名、性别、年龄、身高、体重、体脂率计算 BMI，制定训练强度
健身目标减脂/增肌/体态矫正/体能提升个性化计划方向
身体状况伤病史、禁忌动作安全训练的底线
训练条件经验等级、可用时间、场地、设备匹配可执行的方案
饮食习惯忌口、过敏、饮食偏好饮食方案制定依据

3.2 个性化训练计划生成

教练发送 生成计划 <学员名>，AI 根据学员多维数据自动生成个性化周训练计划，每天包含：热身（5-10min）→ 主训练（动作+组数×次数+休息时间）→ 拉伸（5-10min）。

3.3 运营分析与预警

📈 运营指标

  • 打卡率趋势（周/月）
  • 完成率分布
  • 体重/体脂变化趋势
  • 饮食打卡达标率

🚨 预警机制

  • 🔴 连续缺勤 ≥3 天 → 流失风险
  • 🟡 完成率 < 60% → 计划调整
  • 🟢 续费时机识别 → 商业提醒

3.4 可视化数据看板

基于飞书多维表格，提供学员目标分布（饼图）、打卡热力图（日历视图）、体重变化（折线图）、打卡率排行（进度条）、流失预警（看板视图）等多维可视化。

3.5 定时自动化任务

任务执行时间功能
晨间训练提醒每天 08:55提醒当日有训练计划的学员
晚间打卡检查每天 21:33检查未打卡学员并推送提醒
周度运营报告每周日 20:03自动生成并推送运营数据

4.3 饮食 AI 分析

ClawFit 最具特色的功能。学员只需拍一张餐食照片，AI 即可完成：

  1. 🍽️ 菜品识别 — 自动识别照片中的所有食物
  2. 🔥 热量估算 — 计算总热量（kcal），误差 ±15%
  3. 📊 营养素拆解 — 碳水/蛋白质/脂肪克数与占比
  4. 💬 个性化建议 — 根据学员目标给出针对性建议

05 AI 能力详解

5.1 多模态图片分析

分析类型输入AI 输出应用场景
训练动作分析训练照片动作识别→姿势评估→改进建议→风险提示图片打卡
体重秤 OCR体重秤照片数值提取→单位转换→历史对比→趋势分析体重记录
餐食营养分析餐食照片菜品识别→热量估算→营养素拆解→饮食建议饮食打卡

5.2 自然语言交互

  • 指令式交互：关键词触发特定功能（如「建档」「打卡」「分析」）
  • 自由对话：自然语言描述训练感受、提问健身知识
  • 上下文理解：记住对话历史，提供连贯体验
  • 多角色适配：根据教练/学员身份自动调整风格和权限

5.3 智能提醒系统

基于 OpenClaw Cron 引擎，实现晨间训练提醒（08:55）→ 晚间打卡检查（21:33）→ 周度运营报告（周日 20:03）的三层自动化提醒。

  • checkins/ — 训练打卡记录
  • diet-checkins/ — 饮食打卡记录
  • reports/ — 运营报告

☁️ 飞书多维表格

  • 学员总表
  • 打卡记录表
  • 体重追踪表
  • 饮食记录表

7.2 设计理念

设计决策理由
Markdown 主存储可读性强、版本控制友好、无外部数据库依赖
按月分文件平衡文件大小与查询效率
拼音 ID人类可读、唯一性保证、避免中文路径
图片就近存储与记录同目录，方便关联查看
飞书表格同步提供可视化看板，教练无需离开飞书
